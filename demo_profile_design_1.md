# demo_marketing_prompts 画像设计方案 v2

## 与 v1 的差异

| 维度 | v1 | v2 |
|------|----|----|
| L1 难度动作 | 3 条全用 B4（婉拒义），无区分度 | 婉拒三子类各 1 条：繁忙义、待定义、婉拒义 |
| L2 难度动作 | A1（讨价还价）6 次，动作类型集中 | 去重后保留 3 次 A1，补充提问FAQ、要求跳过验证、不准确信息/自相矛盾 |
| L3 难度动作 | 偏重 C1+C2，缺语用和边界动作 | 补充语义混乱、语言切换、同时和在场者说话 |
| 结构调整 | — | 难度/结局分布、profile 数量不变 |

## 难度定义（三级，与结局无关）

难度是用户在对话全程中的**行为模式**，不影响对话最终走向。L1 可以拒绝（确实不需要，礼貌说不），L3 也可以成交（客服抗住了连珠炮质疑最终说服了用户）。

| 难度 | 行为特征 | 典型表现 |
|------|---------|---------|
| **L1** | 配合顺畅 | 不打断、不追问、听完判断、直接表态 |
| **L2** | 有顾虑、可沟通 | 追问细节、试探边界、讨价还价，消化后能继续 |
| **L3** | 高度对抗 | 打断、质疑合法性/隐私/动机、辱骂、威胁挂断、直接挂断、同 turn 多动作。对话可能极短（开场挂断）也可能极长（连珠炮但不挂） |

## 意愿倾向 × 可转化上限（替代旧版"结局"）

> 对应 gen_v2.md 原则四：旧版用"成交/拒绝/留存"预设结局会让 benchmark 失去对客服行为的敏感度。新版改为两个维度——意愿倾向是用户起点，可转化上限是 benchmark 锚点。

**意愿倾向**（用户起点）：

| 标签 | 含义 |
|------|------|
| `高意愿` | 主动有需求，产品基本符合，轻推即可 |
| `条件意愿` | 有需求但有顾虑，说中利益点才会考虑 |
| `低意愿` | 基本不感兴趣，需要较大努力才能推动 |
| `无意愿` | 完全不适合或不需要，最优客服也不应成交 |

**可转化上限**（benchmark 锚点）：在最优客服表现下，这个用户最好的结局是什么。`无意愿` 类型的可转化上限必须是"无法成交"。

## 设计规则

- 每条 prompt 3 条 profile，共 24 条，难度分布 L1:L2:L3 = 7:10:7（2:3:2）
- 同 prompt 的 3 条意愿倾向各不相同，且覆盖至少 2 种不同的可转化上限
- 必须有至少 1 个 `无意愿` profile 且可转化上限为"无法成交"
- 难度与意愿倾向/可转化上限解耦——L1 可以有无法成交、L3 可以有成交
- **identity 字段**：根据 prompt 占位符填写（参见 gen_v2.md 映射表），不需要则为 `{}`
- **消解风格**（新增）：有对抗触发的 profile（L2/L3 全部 + 含对抗动作的 L1）必须有消解风格；同 prompt 下 3 条 profile 的消解风格不重复；24 条中轻量级≥7、重情绪级≥4、其余四大类各≥1；无对抗 L1 标注 `N/A`（详见 gen_v2.md 原则七）
- **精力基线**（新增）：每条 profile 标注精力基线（高/中/低）。分布规则：高精力 6-7 条（L1高意愿和条件意愿）、中精力 10-12 条（多数 L2）、低精力 5-8 条（L3+低意愿+无意愿）。同 prompt 下尽量不全部同精力基线。精力决定的是"能听多久"——同一套对抗动作，低精力2轮见底，高精力6轮才见底（详见 gen_v2.md Layer 1 精力侧）
