# demo_2_marketing_prompts 画像设计方案 v1

## 与 design_1（v2）的关系

本方案为 demo_2_marketing_prompts.jsonl（8 条新 prompt）的独立画像设计。与 design_1 共享难度定义、意愿倾向/可转化上限框架和设计规则，但 **画像、行为模式、难度动作编码与 design_1 完全不重叠**。

消解风格池同为 18 种——design_1 已全覆盖——本方案复用风格但搭配全新动作组合+画像，整体行为轮廓不重复。

---

## 难度定义（三级，与结局无关）

| 难度 | 行为特征 | 典型表现 |
|------|---------|---------|
| **L1** | 配合顺畅 | 不打断、不追问、听完判断、直接表态 |
| **L2** | 有顾虑、可沟通 | 追问细节、试探边界、讨价还价，消化后能继续 |
| **L3** | 高度对抗 | 打断、质疑合法性/隐私/动机、辱骂、威胁挂断、直接挂断、同 turn 多动作 |

## 意愿倾向 × 可转化上限

| 标签 | 含义 |
|------|------|
| `高意愿` | 主动有需求，产品基本符合，轻推即可 |
| `条件意愿` | 有需求但有顾虑，说中利益点才会考虑 |
| `低意愿` | 基本不感兴趣，需要较大努力才能推动 |
| `无意愿` | 完全不适合或不需要，最优客服也不应成交 |

`无意愿` 类型的可转化上限必须是"无法成交"。

## 设计规则

- 每条 prompt 3 条 profile，共 24 条，难度分布 L1:L2:L3 = 7:10:7（2:3:2）
- 同 prompt 的 3 条意愿倾向各不相同，且覆盖至少 2 种不同的可转化上限
- 必须有至少 1 个 `无意愿` profile 且可转化上限为"无法成交"
- 难度与意愿倾向/可转化上限解耦
- identity 字段：全部 prompt 无模板占位符，统一为 `{}`
- 消解风格：有对抗触发必须有消解风格；同 prompt 下不重复；轻量级≥7、重情绪级≥4、其余四大类各≥1
- 精力基线：高精力 6-7 条、中精力 10-12 条、低精力 5-8 条

### 仍缺失的子类（Design 1+2 均未覆盖）

- D4 矛盾信息陷阱（故意版，如 Tesla 不需要换机油——本方案 #8 为无恶意版）
- E4 缺少必要凭证
- T3 技术能力为零
- T9 非本人账户
- T16 语言混合/语码切换
- F2 全损音质（高频独立出现，非组合）
- 复杂行为轨迹：先拒绝后同意、先同意后拒绝（连续跨 turn 版）

## 优先级

| 优先级 | 范围 | 理由 |
|--------|------|------|
| P0 | 全部 L2（10 条） | 最典型的真实场景，评估区分度最高 |
| P1 | 全部 L3（7 条） | 测客服抗压极限，Design 2 新增 C3/C4/E1/E2/T5/T7/T13/T14/T15 等边界动作 |
| P2 | 全部 L1（7 条） | 基础通过率，Design 2 使用初始沉默、冷淡拒绝、只接受线下、持续拒绝不挂断等新 L1 模式 |
