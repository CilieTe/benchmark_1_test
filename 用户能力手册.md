# 用户能力手册

用户模型（User Simulator）在对话中可用的行为能力参考。写 user profile 的 task_instructions / behavioral_affordances / behavior_examples 时，从这里取材。

**本手册三层结构**：
- **Part A 技术能力** — 用户在对话系统中可以执行哪些动作（`[silence]`、打断、部分回答等）
- **Part B 行为类别** — 这些动作在真实电话中长什么样（四大类 + 真实语料）
- **Part C 引用指南** — 从 gen_v2.md 的对抗类型到本手册的快速映射

# Part A：技术能力定义

用户角色在对话中可用的行为和交互能力。独立于画像的行为模式——画像是"这个人会怎么做"，技术能力是"可以做什么动作"。

## A1. 沉默 `[silence]`

用户在一个 turn 中**要么沉默，要么正常回答，没有中间态**。选择沉默时，content 字段仅放 `[silence]`，不放任何其他文本。

```
{"turn_index": 4, "role": "user", "content": "[silence]"}
```

**含义**：用户听到了上一轮，但选择不回应。不是断线、不是技术故障——就是不想说话。

**禁止的用法**：
- `"[silence] 嗯让我想想..."` — 不存在"先沉默再回答"
- `"嗯...[silence]"` — 不存在"说完再沉默"
- content 中只要不是纯 `[silence]`，就是正常回答

**典型触发场景**：客服报价后用户在消化价格 / 客服连珠炮式追问，用户用沉默施加压力 / 用户已经不想聊了但还没到挂断的程度 / 被动型用户被连续追问后的自然反应

**规范**：
- `[silence]` 只能出现在 user turn 的 content 中，且必须是完整内容
- 连续 `[silence]` 不应超过 2 轮，第 3 轮用户应给出收尾（哪怕是"算了挂了"）

## A2. 打断 `[interrupt]`

用户在 assistant 话没说完时切入。表现为**上一轮 assistant 的 content 末尾**被标记 `[interrupt]`：

```
{"turn_index": 3, "role": "assistant", "content": "我们现在有个特别优惠，每月只需299，而且还赠送首月[interrupt]"}
{"turn_index": 4, "role": "user", "content": "等一下，299是含税还是不含税？"}
```

**典型触发场景**：客服说到价格/敏感信息，用户立即反应 / 客服绕弯子，用户不耐烦切入 / 听到了足够的信息不想听完 / 客服说错关键信息，用户立即纠正

**规范**：
- `[interrupt]` 标记在 assistant turn 的 content 末尾
- 对话生成时 assistant 正常输出完整内容，生成后再截断加标记
- user 下一轮 content 应对齐打断点——用户只听到 `[interrupt]` 之前的内容
- 一个 turn 最多被打断一次

## A3. 不配合 / 拒绝回答

用户有权在任何时候不配合客服的引导。不配合 ≠ 对抗，是一个宽频谱：

| 子类型 | 说明 | 例句 |
|--------|------|------|
| 直接拒绝 | 明确说不想回答 | "这个我不太想说" |
| 回避/转移 | 把话题引开 | "哎你先告诉我你们那个299到底快不快" |
| 模糊回应 | 答了但没提供信息量 | "不一定，看情况吧" |
| 反问 | 把球踢回去 | "你先说你们最低多少钱吧" |
| 给假信息 | 方便借口，不真实但合逻辑 | "我微信不怎么用，算了吧" |

**设计原则**：不配合的程度和方式由 user profile 的对抗类型决定——沉默被动型用模糊回应，专业质疑型用反问，话多绕弯型用回避/转移。不要让人物违背自己的行为模式。

## A4. 自然语用纠错

真实用户纠正对方错误时不说"你说错了"——把纠正嵌入自然语流中。

```
错误：user: "你搞错了，我不是张女士，我是张先生。"    ← 元对话，太干净

正确：user: "啊？我是张先生，不是张女士。不过尾号4827没错，是我本人。"
```

**关键原则**：
- 不用"你说错了""不对""搞错了"评价对方
- 纠正信息本身，不评价对方行为
- 纠正完立刻回到正题，不让对话卡在纠错上
- 语气取决于人物设定

| 场景 | 生硬版本 | 自然版本 |
|------|---------|---------|
| 名字叫错 | "你叫错了，应该是王总" | "叫我老王就行" |
| 产品说错 | "你说错了，我要的是A" | "哎不对，我说的是A——没事你继续" |
| 价格报错 | "你说的价格不对" | "嗯？上次跟我说的是199啊？" |
| 称谓用错 | "请不要叫我女士" | "不用那么正式，直接叫我名字就好" |

> 参考语料：Part B § B1 纠错型

## A5. 回合级交互行为

### 挂断 / 结束对话
- **硬挂断**："不说了，挂了"——对应 `[end_call]` 或直接结束对话序列
- **软退场**："行那我先忙了"、"好的我再考虑考虑，先这样"
- **承诺退场**："等我问一下我老公/等我看了再联系你"

### 迟疑 / 停顿
内容层面填充词（"嗯……""那个……""让我想想"），或简短确认但无推进（"嗯""哦""这样啊"）。这些不是 `[silence]`——用户说了话，但没推进对话。

### 部分回答
客服问了 A 和 B 两个问题，用户只回答 A，忽略 B。真实用户几乎从不完整回答多问句。

> 客服：您目前用什么套餐，一个月大概花多少钱？
> 用户：我现在用的移动的套餐。        ← 只答了运营商，没答价格

### 后知后觉
过了两三轮才反应过来前面某个信息有问题，回过去追问。打破线性对话的假设。

> （3 轮后）
> 用户：哎等一下，你刚才说299是一个月还是一年？

> 参考语料：Part B § B3 等一下回溯

## A6. 能力与画像的映射

能力库是"菜单"，用户画像是"点菜"。不是所有能力每个用户都用。

| 能力 | 适用画像类型 | 禁用场景 |
|------|------------|---------|
| `[silence]` | 沉默被动型、说不清楚型 | 话多绕弯型（沉默不了） |
| `[interrupt]` | 专业质疑型、价格敏感型 | 第三方依赖型（等人说完） |
| 拒绝回答 | 全部类型，表达方式不同 | — |
| 自然纠错 | 全部类型 | — |
| 挂断 | 无意愿型（硬挂）、低意愿型（软退场） | 高意愿型不应主动挂断 |
| 部分回答 | 话多绕弯型、专业质疑型 | — |
| 后知后觉 | 反复横跳型、说不清楚型 | 专业质疑型（应是精确的） |

# Part B：四大行为类别 × 真实语料

来自 filtered_data_0514_cleaned.jsonl（10238 条电话营销/催收对话）的手工筛选样本。每条标注了行为模式和为什么自然。

> 数据以西班牙语为主（拉美市场），印尼语/他加禄语次之。中文示例为构造。

## B1. 变形确认（Identity Blurring）

承认证实身份，但不给标准的"对/是的"。在确认身份的同时修正错误、设限、或反问。

### 纠错型 — 承认是本人，顺手纠正对方错误

**#1** 纠正性别（最典型的纠错模式）
```
[客服] Necesito confirmar si hablo con MANUEL RAMIREZ. ¿Es usted?
[用户] no ya dije que no no no no no hay soy mujer cómo va a ser manuel rammiro
```
> "我是女的，怎么可能是Manuel Ramirez"——用身份事实让客服自己推导出错误。

**#2** 用声音证据纠正
```
[客服] ¿Tengo el gusto con MARIA DEL ROCIO MORA?
[用户] bueno no si me oyes el la voz soy hombre
```
> "你听我声音，我是男的"——用感知证据而非抽象陈述来纠正。

**#3** 直接报名字纠正
```
[客服] En el sistema tenemos registrado este número con el nombre de GRISELDA CONTRERA ESTRADA, ¿es usted?
[用户] no yo me llamo roberto yo soy es mero que yo soy hombre no soy mujer
```
> 三重纠正：名字+性别+否定。

**#4** 中文对照（构造）
```
[客服] 您好，请问是张女士吗？我是ABC银行的大卫·李。
[用户] 啊？我是张先生，不是张女士。不过尾号4827没错，是我本人。
```
> 纠正 + 确认，一个语流完成，纠正完立刻回正题。

### 占位型 — 承认是本人，立刻设限

**#1** 在开车
```
[客服] Le llamo porque hoy tenemos una oferta exclusiva. ¿Me permite contarle de qué se trata?
[用户] no pues ahorita voy manejando si quieres después por favor porque ando manejando ahorita
```
> "我在开车"——最不可辩驳的占位理由。

**#2** 在开车 + 危险警告（升级版）
```
[客服] Le llamo porque hoy tenemos una oferta exclusiva...
[用户] dijo si me hablas en la tarde ando acá en estadoeu unidos y ando manejando y acá es muy peligroso
```
> 叠加安全理由，让对方不敢继续占用。

**#3** 在工作（最短版）
```
[客服] Le llamo porque hoy tenemos una oferta exclusiva y está buenísima para ahorrar...
[用户] ah estoy ocupada amigo es estoy trabajando
```
> "我是，但我在工作"——一句话确认+设限。

### 反问型 — 不直接承认，先反问

**#1** 最简短反问
```
[客服] Muy buenas tardes ROSA CLEOTILDE CALDERON.
[用户] qué necesitaba
```
> "你要干嘛"——连"Hola"都不回。

**#2** 问来源
```
[客服] Buenas tardes, le habla Sofía Martínez de ízi. ¿Se encuentra Martín López Hernández?
[用户] quién habla dígame
```
> "你谁，说"——经典反问组合。

**#3** 声明不接陌生电话
```
[客服] Buenos días, ¿tengo el gusto con señor Christian?
[用户] quién habla no contestado teléfono extraños
```
> 反问 + 规则声明。

**#4** 中文对照
```
[客服] 您好，请问是李明先生吗？
[用户] 哪位？什么事？
```
> 不承认身份的批量处理方式——先确认对方再决定是否暴露自己。

## B2. 跳级推进（Intent Leaping）

不说赞同词，直接越过客套切入实质利益、算账或质疑。

### 算账型 — 脑子里直接换算

**#1** 纠正报价
```
[客服] Actualmente su plan tiene un costo de 680 pesos al mes. Con la mejora sería de 780 pesos.
[用户] mmm no es que pago ocho noventa pago pa pago novecientos pesos pago mucho
```
> "我付的是八九百"——用户知道实际账单，不是你说680就680。

**#2** 心算叠加
```
[用户] pago novecientos y si ya sé y si ya me subes cien pesos tú voy a pagar mil pesos dos
```
> "900 + 100 = 1000多"——已经算完了客服要说的。

**#3** 追问涨价时间线
```
[客服] Aumentamos su internet a 300 Megas y le incluimos bonificaciones...
[用户] pero cuándo empezaría a pagar esos cien pesos
```
> 不问"能不能便宜"，问"什么时候开始扣"——默认接受了涨价但想控制时间。

**#4** 中文对照
```
[客服] 这个计划每月只需存入2000元，银行还会额外匹配5%的种子基金。
[用户] 那我每个月存2000，你们月底能直接给我卡里打100块吗？
```
> 直接换算成到账金额，跳过"这个产品不错"的表态。

### 疑虑先行型 — 用防御性提问表达关注

**#1** 担心额外费用
```
[客服] ...con el otro no tenía un costo extra...
[用户] gracias muy amablezo pero como le digo tiene un costo extra lo que me está diciendo joven con el otro no tenía un costo extra nomás pagaba lo que era lo que estoy pagando con ustedes es mucho
```
> "你们有额外费用，别家没有"——不说不付，说的是"你们比别家贵"。

**#2** 先质疑基础服务质量
```
[客服] ...le ofrezco la activación gratis además de los descuentos por 300 pesos anuales...
[用户] sí no mire si va a ser si va a ser así lento pues no tiene caso de que me esté aumentando más el internet yo iba a ir a hablar allá para que lo voy a cancelar o me lo componen o ya no tengo internet porque realmente está muy lento
```
> 不拒绝升级，但先质疑基础服务。"你要我多付钱，但现有的都不好用"。

**#3** 中文对照
```
[客服] 这个计划不会有任何隐藏费用，随时可以取消。
[用户] 听着倒还行，但这不会是那种捆绑消费吧？后期收不收费？
```
> 本质上感兴趣，但先用防御性提问保护自己。

### 场景带入型 — 立刻代入自己的生活验证

**#1** "我在家，不需要高速"
```
[客服] ...el costo neto en realidad es mínimo...
[用户] mira te voy a decir no no aceptó eso porque todo el tiempo estoy aquí en casa o sea todo eso que tú me dices es para alguien que trabaja yo soy en una persona pensioda que está en casa o sea no necesito nada de eso
```
> "我是退休在家的，你说的都是给上班族用的"——把产品价值在自己场景里归零。

**#2** "我和老婆都不在家"
```
[客服] ...esta mejora no tiene costo de activación...
[用户] porque no vemos mi esposa y yo no vemos película nada nos vamos a trabajar regresamos ya tarde los dos nos vemos un ratito merendamos algo a dormir y otra vez a trabajar
```
> 详细描述生活方式，证明产品没有位置。

**#3** 中文对照
```
[客服] 这个自动记账功能可以帮你分类每月开销。
[用户] 那它能自动识别我点外卖的账单吗？
```
> 立刻代入自己的高频场景验证。

## B3. 异步处理（Asynchronous Cognitive Drift）

对话推进到步骤C时，突然回退追问步骤A的细节。

### "等一下"回溯 — 滞后反应，往回拉

**#1** 价格回溯
```
[客服] ...pagará 680 pesos, que es su paquete actual más 50 pesos...
[用户] cuánto pagaría por todo no fíjate que pago pago con pa oye no no lo no lo que pasa es que pago trescientos y feria
```
> 客服报680，用户脑子转了一圈发现不对——"等等，我实际付的是300多"。

**#2** 记账逻辑追溯
```
[客服] ...¿en qué puedo apoyarle?
[用户] oiga lo que pasa es que me ya me han estado marcando señorita eh eh ya lo pagué yo pagué trescientos ochenta y nueve pesos ahora usted me dice que es otra cantidad por eso digo ese eso que me están cobrando ahorita es por no pagarlo luego o o cómo es
```
> "你说另一笔钱——是不是因为没及时付产生的？"——过了一两轮才反应过来追问计费逻辑。

**#3** 中文对照
```
> （3 轮后）
> 用户：哎等一下，你刚才说299是一个月还是一年？
```

### 关键词跑题
> 本数据集路径较紧，自然跑题样本少。待补充更开放的数据源。

## B4. 软性消解（Soft Repulsion）

不用冷冰冰的"不需要"，用第三方/时间/空间把推销价值消解掉。

### 伪需求消解 — 证明已有同类更好的方案

**#1** 已有其他服务商
```
[客服] Disculpa, no puedo escucharte bien. ¿Podrías repetir?
[用户] bueno ya tengo este los servicios y ya lo tengo también el paquete entonces no estoy interesado muchas gracias
```
> "我已经有服务了"——消解对方存在价值。

**#2** 已有类似产品
```
[客服] ...Con el beneficio de 300 Megas el costo neto en realidad es mínimo...
[用户] no gracias ya cuento con bueno con algo así similar
```
> 不说具体，但让对方卖点失效。

**#3** 中文对照
```
[客服] 现在预约空调清洗有八折优惠。
[用户] 不用了，我女婿就是开家电清洗店的，上周刚帮我洗过。
```
> 你的价值在我的社会关系网里等于零。

### 时间/空间错位 — 用客观行程拒绝

**#1** 不在城里
```
[客服] ...hoy le puedo ofrecer duplicarle la velocidad a su servicio, por un costo de solo 50 pesos.
[用户] eh por lo pronto no porque ando fuera de la ciudad voy a tardar como dos tres semanas
```
> "我出城了，要两三周"——不是拒绝，是"物理上不可能现在处理"。

**#2** 到周末才在
```
[客服] ...Le invito a regularizar su servicio con el pago de setecientos diecinueve pesos el día de hoy o a más tardar mañana.
[用户] sí pues claro que sí nomás dije que venga ya paso pasar hacer hacerle ese comentario porque yo que desconozco casi no estoy aquí salgo fuera y no nomás llegó fines de semana
```
> 把deadline推到自己的时间。

**#3** 中文对照
```
[客服] 方便本周来店里了解一下吗？
[用户] 我明天就要去国外出差两个月，等我回来自己去小程序上看吧。
```

### 主权转交 — 决策权推给不在场的第三方

**#1** 推给配偶
```
[客服] ...¿Le gustaría que le active la actualización?
[用户] déjame hablar con mi esposo a ver qué me dice sí y luego le confirmo
```
> 不说不，但切断当场决策回路。

**#2** 推给儿子（付款人分离）
```
[客服] ...¿Lo activamos?
[用户] tú contigo hay comparar sinz le vuelvo a decir mi hijo es el que lo paga yo no puedo pagar
```
> "我儿子付钱，我做不了主"。

**#3** 推给爸爸
```
[客服] ...¿Realiza el pago hoy?
[用户] sí pues ahorita deja le marco a mi papá y a ver qué me dice vale
```
> 明显的延迟策略。

**#4** 推给律师
```
[客服] ...Le llamamos para apoyarle con la reactivación...
[用户] mire señorita ya no con las amenazas que ustedes están dando por teléfono no ya no quiero nada para quen amenazando de lo que ya comenté yo con mi familia licenciado...
```
> "我跟我的律师聊过了"——把主权重定向到法律层面。

**#5** 中文对照
```
[客服] 今天确认的话可以锁定优惠价格。
[用户] 这事得我老婆管，家里钱都在她那，你跟她说吧。
```

| 价格敏感型 | B2 跳级推进-算账型 / B4 伪需求消解 | 反问、直接拒绝回答 |

## C2. 写 task_instructions 时的取用顺序

1. 确定用户的对抗类型（→ gen_v2.md 8 种）
2. 查 C1 映射表，找到对应的首选行为类别
3. 读 Part B 中该类别的真实语料，感受语言风格和情绪强度
4. 从 Part A 中选择 1-2 个技术能力，填入动机层 2-3 的触发条件
5. 写 behavior_examples 时，参考 Part B 语料的句式/长度/情绪

## C3. 写 behavioral_affordances 时的取用顺序

1. "你倾向于用以下方式表达不满或顾虑"：从 Part B 对应类别选 2-3 个具体动作
2. "你不会做的"：从 Part A 能力映射表（禁用场景列）中排除
3. "你的委婉表达习惯"：低意愿/无意愿类从 B4 软性消解中取材

## C4. 语言归纳

| 行为 | 西班牙语（本数据主力） | 印尼语 | 中文等效 |

| 反问型确认 | `¿quién habla?` `¿de dónde llama?` | `dari mana` `siapa ini` | 哪位？什么事？ |

| 纠错式确认 | `soy hombre/mujer` `no soy [nombre]` | — | 我是张先生，不是张女士 |
